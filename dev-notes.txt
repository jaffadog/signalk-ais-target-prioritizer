
src/
├── assets/             # Images, fonts, and global CSS
├── components/         # Reusable UI components (Buttons, Inputs, etc.)
│   ├── Button/
│   │   ├── Button.jsx
│   │   ├── Button.css
│   │   └── index.js    # For cleaner imports
├── hooks/              # Custom React hooks
├── pages/ (or views/)  # Main page-level components (Home, Profile)
├── utils/ (or helpers/)# Generic JS helper functions (formatting, validation)
├── services/           # API calls and external integrations
├── App.jsx             # Main application component
└── main.jsx            # Entry point

Naming Conventions:

PascalCase for components and their files (e.g., Header.jsx).
camelCase for helper files and general utility functions (e.g., dateHelpers.js).

The index.js Pattern: Creating an index.js file inside a component folder 
allows you to import it more cleanly: import { Button } from './components/Button' 
instead of ./components/Button/Button.
